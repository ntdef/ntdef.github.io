Sometimes in a moment of weakness I'll wander about to math/programming puzzle page and try my luck at the newest posting. I've been working on slogging through the Project Euler problems, but every now and then I'll check PuzzlOR, a website dedicated to puzzles relating to [operational research] and logistics. The puzzles there range from medium difficulty to almost trivial (i.e. you don't really need a computer to solve the puzzle).

This month's problem lies among the trivial--- but hey, for the chance to win \$25? Heck yeah. Basically, the question is how do you order a set of 3 pennies, 3 nickels and 3 dimes such that they sum across the rows and columns equal to the values in the pictures below.

How many permutations do we need to try?

The first question is, how difficult is this problem going to beto compute? As it turns out, not very: to compute the number of $n$ permutations of $k$ items, we have

$$ \frac{n!}{n_1!n_2!n_3!...n_k!} $$

In this case, we have 3 distinct items, each with 3 elements. So the total possibility space is $\frac{9!}{3!^3} = 1680$ permutations.

We can verify that this is true by generating all possibilities.

Since the number of things we need to try are so few, we can just brute force our way to a solution. Now to find our solution, we'll turn every permutation into a matrix and search the collection of matrices until we find one that meets our criteria.

Its simplicity makes it suitable for small projects where you want to get to an answer quickly. Now we can model the behavior of the frog and the fly as two parallel random walks. We'll start with giving the starting position, and then work our way forward by choosing a random node to move to next among the neighbors, with equally weighted probabilities.

In [145]: 
FROG_POS = 'B2'
FLY_POS = 'C3'

def next_position(graph, cell):
    return random.choice(graph.neighbors(cell))

We'll keep running our simulation until the position of the frog is the same as the position of the fly, and then return that cell.

In [162]: 
def simulate(frog_pos, fly_pos):
    while frog_pos <> fly_pos:
        frog_pos = next_position(frog, frog_pos)
        fly_pos = next_position(fly, fly_pos)
    return frog_pos

Let's simulate this event 1,000,000 for good measure.

So it looks like B1 was the clear winner by quite a margin. But simulations can seem like such a cop out. Is there any way to tell a priori what the highest probability collision cell might be? Luckily, with a little probability theory and markov chains, there is.

First, lets conver our grap to a stochastic matrix. A stochastic matrix is a matrix represenation of a markov chain where the entries represent the probability of going from state i to state j. In our case, the stochasic matrix represents the probability of our critter moving from a cell to any other cell.

We'll write a function that will give equal weights to moving over each edge of our graph and return the desired matrix.

In [324]: 
def convert_to_stochastic(graph):
    new_graph = nx.DiGraph()
    new_graph.add_nodes_from(graph)
    for node in graph.nodes():
        n = graph.degree(node)
        for nbr in graph.neighbors(node):
            new_graph.add_edge(node, nbr, weight=1.0/n)
    return nx.to_numpy_matrix(new_graph, sorted(new_graph.nodes())).T

# SO note that for this to make sense, you need to make it geometric. 
# The intersection at time t means time at first intersection, which means you have to 
# compute the probability of not intersecting until time t, which is a geometric random variable.

Let $Y_{ij}(t)$ be the probability of the fly being at the $ij$-th cell at time t, and let $X_{ij}(t)$ be likewise for the frog for $i$ in $\{A,B,C\}$ and $j$ in $\{1,2,3\}$. Let $p$ be a vector where the $i$th entry is 1 for the starting cell of the critter and 0 otherwise, and let $M$ be the stochastic transition matrix. We can compute the probability of being in the $ij$th cell at time $t$ is given by the $ij$the entry of the vector $q(t)$, where $$q(t) = M^{t}p$$.

Since $X$ and $Y$ are independent, the probability of both being at the $ij$ cell at time $t$ is $Z_{ij}(t) = X_{ij}(t)Y_{ij}(t)$ and we have $$ Z_{ij}(t) = q_X(t)q_Y(t)$$.
